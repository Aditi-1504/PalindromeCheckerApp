# PalindromeCheckerApp

---

## Running Procedure

1. Compile the program
   `javac UseCase1PalindromeCheckerApp.java`

2. Run the program
   `java UseCase1PalindromeCheckerApp`

---

## Flow of Project

* Application starts.
* JVM calls the `main()` method.
* Welcome message is displayed.
* Application name and version are shown.
* Program proceeds to palindrome checking use cases.

---

## Objective

The objective of the **PalindromeCheckerApp** is to design and implement a console-based Java application that checks whether a given string is a palindrome using multiple approaches.
The project also helps strengthen **core Java programming concepts and data structure understanding**.

## Use Cases

### UC1 – Application Entry & Welcome Message

Displays the application name and version when the program starts.

### UC2 – Print a Hardcoded Palindrome Result

Checks palindrome using a predefined string.

### UC3 – Palindrome Check Using String Reversal

Reverses the string and compares it with the original string.

### UC4 – Character Array Based Palindrome Checker

Uses a character array to compare characters from both ends.

### UC5 – Stack-Based Palindrome Checker

Uses a stack data structure to validate palindrome strings.

### UC6 – Queue + Stack Based Palindrome Checker

Uses both queue and stack to compare characters.

### UC7 – Deque-Based Optimized Palindrome Checker

Uses a double-ended queue for optimized palindrome validation.

### UC8 – Linked List Based Palindrome Checker

Checks palindrome using a linked list structure.

### UC9 – Recursive Palindrome Checker

Uses recursion to compare characters from both ends.

### UC10 – Case-Insensitive & Space-Ignored Palindrome Checker

Ignores case differences and spaces while checking palindrome.

### UC11 – Object-Oriented Palindrome Service

Implements palindrome logic using OOP principles.

### UC12 – Strategy Pattern for Palindrome Algorithm

Uses the Strategy design pattern to support multiple algorithms.

### UC13 – Performance Comparison

Compares execution performance of different palindrome approaches.
